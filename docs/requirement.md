# OA报销AI智能体系统 — 需求文档

## 1. 项目背景

企业日常报销流程依赖人工录入发票、行程单信息，效率低、易出错。本项目基于开源智能体编排平台 LangGraph 与 DeepSeek 大模型，与 OA 系统集成，实现报销单据的智能识别、自动校验与流程流转，提升报销处理效率与合规性。

## 2. 目标用户

- **普通员工**：提交报销申请
- **审批领导**：审核报销单据（支持驳回 / 通过 / 转审）
- **财务人员**：复核与归档
- **系统管理员**：规则配置、用户角色管理、审计日志查看

## 3. 功能需求

### 3.1 阶段一：申请与预处理

| 编号 | 需求 | 优先级 |
|------|------|--------|
| R1.1 | 用户可扫描或上传发票、行程单等报销凭证图片/PDF | P0 |
| R1.2 | 用户填写基本报销详情（申请金额、事由、申请日期等） | P0 |
| R1.3 | 系统接收图片与单据，自动形成并启动报销工作流程 | P0 |
| R1.4 | 系统支持多文件并发上传与批量校验（前端拖拽多选，后端并发处理） | P1 ✅ 已实现 |

### 3.2 阶段二：AI 智能读取与校验

> **校验执行顺序**：OCR 提取 → 异常检测（前置拦截）→ 分类限额校验。异常检测若拦截则直接返回，跳过后续校验。

| 编号 | 需求 | 优先级 |
|------|------|--------|
| R2.1 | 系统通过 API 调用 DeepSeek AI Agent 进行图像识别与 OCR，提取发票代码、日期、金额等关键字段 | P0 |
| R2.2 | 系统通过 NLP 与数据语义分析，识别费用类型（差旅/餐饮/住宿/交通/办公/其他） | P0 |
| R2.3 | 系统连通外部 API（增值税发票查验平台、银行流水）进行真实性核验 | P1 |
| R2.4 | 系统根据公司制度规则集进行核对（金额限额、费用分类、审批权限） | P0 |
| R2.5 | 系统进行异常检测（前置拦截），识别字段缺失、格式错误、重复报销、金额异常（含申请金额不足）、票据过期、票据即将过期（剩余有效期<30天预警）等风险 | P0 |
| R2.6 | AI Agent 返回校验结果：通过 / 预警 / 拦截 / 错误（AI 调用失败等系统异常，转人工兜底） | P0 |
| R2.7 | 系统支持小额免审：发票金额 ≤ 100 元跳过分类限额校验 | P1 |
| R2.8 | 系统支持行程单 OCR 提取与合理性校验（行程数、金额、时间等） | P1 ✅ 已实现 |

### 3.3 阶段三：流转与审批

| 编号 | 需求 | 优先级 |
|------|------|--------|
| R3.1 | 系统自动填充报销单字段（日期、金额、摘要、费用分类、发票号码、销售方名称、税额等） | P0 |
| R3.2 | 系统显示 AI 校验意见与预警标识（费用超限、重复报销、票据过期、金额异常、字段缺失） | P0 |
| R3.3 | 报销单按金额阶梯审批流转（≤5000元直属领导→财务；≤20000元部门总监→财务；≤100000元VP→财务；>100000元CEO→财务），金额≥50000元需两人会签，支持驳回与转审 | P0 |
| R3.4 | 审批完成后，系统进行数据归档（保留≥7年）与费用发放 | P1 |
| R3.5 | 前端展示执行流水线可视化（LangGraph 节点逐步执行动画：路由→OCR→异常检测→分类/校验，含节点名称、调用工具及执行详情） | P1 ✅ 已实现 |

## 4. 非功能需求

| 编号 | 需求 | 说明 |
|------|------|------|
| N1 | 性能 | 单张票据 AI 识别与校验用户感知响应时间 ≤ 10 秒（异步处理，用户无需等待） |
| N2 | 准确率 | 发票关键字段提取准确率 ≥ 95% |
| N3 | 可用性 | 系统可用性 ≥ 99.5% |
| N4 | 安全性 | 敏感数据（发票信息、金额）传输与存储加密（AES-256），敏感字段脱敏展示 |
| N5 | 扩展性 | 支持后续新增票据类型（火车票、机票等），插件化 Schema + 配置驱动规则 |

## 5. 集成需求

- **OA 系统**：流程启动、表单填充、审批流转、数据归档（见设计文档 §1.3、§14 OA 适配器层）
- **LangGraph**：智能体编排平台，StateGraph 工作流编排（票据路由 → 发票/行程单双 Agent 分支，见设计文档 §16）
- **DeepSeek AI Agent**：OCR 识别、语义分析、智能校验
- **增值税发票查验平台**：发票真实性核验
- **数据库**：当前 SQLite（功能验证），目标 MySQL 8.0（生产环境，见设计文档 §1.3、ADR-005）

> 重大技术决策记录（ADR-001 至 ADR-008）见设计文档 §10。
