# 报销智能化系统 — 需求文档

## 1. 项目背景

企业日常报销流程依赖人工录入发票、行程单信息，效率低、易出错。本项目基于 DeepSeek AI Agent 与 OA 系统集成，实现报销单据的智能识别、自动校验与流程流转，提升报销处理效率与合规性。

## 2. 目标用户

- **普通员工**：提交报销申请
- **审批领导**：审核报销单据
- **财务人员**：复核与归档

## 3. 功能需求

### 3.1 阶段一：申请与预处理

| 编号 | 需求 | 优先级 |
|------|------|--------|
| R1.1 | 用户可扫描或上传发票、行程单等报销凭证图片/PDF | P0 |
| R1.2 | 用户填写基本报销详情（申请金额、事由等） | P0 |
| R1.3 | 系统接收图片与单据，自动形成并启动报销工程流程 | P0 |

### 3.2 阶段二：AI 智能读取与校验

| 编号 | 需求 | 优先级 |
|------|------|--------|
| R2.1 | 系统通过 API 调用 DeepSeek AI Agent 进行图像识别与 OCR，提取发票代码、日期、金额等关键字段 | P0 |
| R2.2 | 系统通过 NLP 与数据语义分析，识别文本上下文（如业务旅行、餐饮等分类） | P0 |
| R2.3 | 系统连通外部 API（增值税发票查验平台、银行流水）进行真实性核验 | P1 |
| R2.4 | 系统根据公司制度规则集进行核对（金额限额、费用分类、审批权限） | P0 |
| R2.5 | 系统进行异常检测，识别重复报销、金额异常、票据过期等风险 | P1 |
| R2.6 | AI Agent 返回校验结果：通过 / 预警 / 失败 | P0 |

### 3.3 阶段三：流转与审批

| 编号 | 需求 | 优先级 |
|------|------|--------|
| R3.1 | 系统自动填充报销单字段（日期、金额、摘要、费用分类等） | P0 |
| R3.2 | 系统显示 AI 校验意见与预警标识（警告/警用标/费用超标） | P0 |
| R3.3 | 报销单按预设流程流转审批（直属领导审批 → 财务审核） | P0 |
| R3.4 | 审批完成后，系统进行数据归档与费用发放 | P1 |

## 4. 非功能需求

| 编号 | 需求 | 说明 |
|------|------|------|
| N1 | 性能 | 单张票据 AI 识别与校验响应时间 ≤ 10 秒 |
| N2 | 准确率 | 发票关键字段提取准确率 ≥ 95% |
| N3 | 可用性 | 系统可用性 ≥ 99.5% |
| N4 | 安全性 | 敏感数据（发票信息、金额）传输与存储加密 |
| N5 | 扩展性 | 支持后续新增票据类型（火车票、机票等） |

## 5. 集成需求

- ** OA 系统**：流程启动、表单填充、审批流转、数据归档
- **DeepSeek AI Agent**：OCR 识别、语义分析、智能校验
- **增值税发票查验平台**：发票真实性核验
- **银行系统 API**：流水核对（可选）
