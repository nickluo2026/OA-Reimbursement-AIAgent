# ADR-006：票据文件存储策略（本地 → 对象存储 OSS）

- 状态：已采纳
- 日期：2025-07（迁移目标 V2.0，当前 V1.4 仍暂存本地）
- 关联：constitution.md §2.3；design.md §1.3、§2.1.2、§7.1.1；README §八
- 作者：架构评审（EA）

## 背景

票据图片/PDF 含发票号码、金额、税号等敏感信息，按宪章 §2.3 须 AES-256 加密存储，
且报销凭证需保留 ≥7 年（税务合规）。当前 V1.4 仅暂存本地 `uploads/` 临时目录，
无加密、无多副本、无生命周期管理，不可用于生产。

## 方案

- **A. 本地文件系统**：零依赖，但无加密、无冗余、单机故障即丢失。
- **B. 对象存储 OSS（AES-256 服务端/客户端加密 + 多副本 + 冷存储分层）**：满足加密、
  持久化、7 年归档与跨可用区冗余，契合部署架构 §7.1。
- **C. 数据库 BLOB**：与业务库耦合，大文件拉低 DB 性能，不推荐。

## 决策

选择 **方案 B** 作为生产目标；V0.x/V1.4 阶段以本地 `uploads/` 临时存储加速验证，
目标 V2.0（生产化）迁移至 OSS。迁移通过存储适配器封装，业务代码仅依赖抽象接口。

## 影响

- 新增存储适配器层，当前本地实现、目标 OSS 实现可切换（对齐 §14 OA 适配器模式）；
- 归档阶段票据落 OSS 低频/冷存储，7 年期满按 NIST 800-88 安全擦除（design.md §15.4）；
- 风险：OSS 迁移需改造文件读写路径与归档流程，已在 §7.1.1 迁移计划中列管（V2.0 对接 OSS，AES-256 加密）。
