├── skill/                    # 核心技能包
│   ├── agent.py              # 编排入口（委托 StateGraph）
│   ├── workflow.py           # 审批/财务工作流

│   ├── config.py             # 配置加载（YAML + 管理员覆盖）

2.3 代码评审结论

  • 架构设计优秀：LangGraph StateGraph 实现节点解耦，条件边路由（OCR失败早退、异常拦截跳过、小额免审）清晰；Agent 注册中心支持插件化扩展。
  • 分层职责清晰：tools（原子能力）/ nodes（图节点+持久化）/ agents（Agent 编排）/ orchestrator（图定义）四层分离，单一职责。
  • 规则与代码分离：费用限额、异常规则、审批权限均通过 YAML 配置，管理员可在 system_config 表覆盖，无需改代码。
  • 持久化设计合理：SQLAlchemy ORM 9 张表（员工/报销单/发票/历史/审批/AI校验/系统配置/审计/用量），索引与外键关系完整。
  • 异常处理健壮：所有持久化操作 try/except 包裹且标注「非致命」，DeepSeek 调用失败有规则引擎兜底，OCR 扫描件自动降级 Vision API。
  • 日志可追溯：structlog 结构化日志 + request_id 上下文追踪 + 审计日志表（仅追加不可删），符合合规要求。
  • 无 Linter 错误：pyright 静态检查 0 错误，代码风格统一（black/ruff 配置完备）。
  • 演示密码统一为 "123456"：已用 PBKDF2/scrypt 哈希存储（非明文），但生产环境必须替换为真实密码库（已在代码注释 [S-001] 标注）。
