# OA报销AI智能体系统 
让报销从"填表"变为"确认"，将人工处理时间降低 80% 以上。

## 项目背景
企业日常报销流程依赖人工录入发票、行程单信息，效率低、易出错。本项目基于智能体编排平台LangGraph和DeepSeek大模型，使用多个智能体对不同类型的票据进行智能识别、异常检测、分类限额、合规性校验等，提升报销处理效率与合规性。

## 一、系统解决什么问题？

### 传统报销的痛点

| 痛点 | 现状 | 影响 |
|------|------|------|
| 人工录入慢 | 员工手动抄写发票号码、金额、日期、税号 | 单张发票录入需 3-5 分钟 |
| 容易出错 | 手动输入金额/号码易抄错 | 财务退回反复修改，周期拉长 |
| 合规风险 | 重复报销、过期发票、金额超标难以人工发现 | 财务审计风险 |
| 审批效率低 | 审批人无法快速判断票据是否合规 | 审批积压，报销周期长 |

### 本系统的价值

- **自动识别**：上传发票/行程单图片，AI 自动提取全部字段，无需手动录入
- **智能校验**：自动检测重复报销、过期票据、金额异常等风险，拦截不合规报销
- **合规保障**：按公司制度自动校验费用分类限额，超限自动预警
- **审批加速**：AI 校验结果直接呈现给审批人，辅助快速决策
- **全程可追溯**：每一步 AI 校验结果留档，支持事后审计

## 三、报销处理全流程

### 员工视角：从提交到拿款的完整旅程

```
第①步          第②步           第③步            第④步              第⑤步          第⑥步
上传票据  →  AI自动识别  →  AI智能校验  →  审批流转（按金额）  →  财务复核  →  费用发放
                  ↓                ↓                  ↓
            提取发票信息     检测异常风险      领导/总监/VP/CEO
            （号码/金额      （重复报销/       按金额阶梯审批
             /日期/税号       过期/超限等）    ≥5万需两人会签
             /商品明细等）
```

### AI 校验的三步流程（员工无感知，10 秒内完成）

| 步骤 | 做什么 | 结果 |
|------|--------|------|
| **第1步：智能识别** | AI 读取发票/行程单，提取发票号码、日期、金额、税号、销售方、商品明细等全部字段 | 生成结构化数据，无需人工录入 |
| **第2步：异常检测** | 自动检查字段是否缺失、发票是否重复报销、是否过期、金额是否异常、申请金额是否充足 | 发现严重问题直接拦截，退回员工修改 |
| **第3步：分类限额** | AI 自动识别费用类型（餐饮/交通/住宿/差旅/办公/其他），校验是否超过分类限额 | 超限标记预警，提醒审批人注意 |

> 100 元以下的发票自动跳过分类限额校验（小额免审），加快处理速度。

## 四、校验结果说明

每张票据提交后，系统会给出以下四种结果之一：

| 结果 | 含义 | 后续动作 |
|------|------|---------|
| **通过** | 票据合规，无异常 | 自动进入审批流程 |
| **预警** | 存在轻微风险（如费用超限、票据即将过期） | 仍进入审批流程，但标注预警提醒审批人注意 |
| **拦截** | 存在严重问题（如重复报销、金额异常、字段缺失） | 直接退回员工，标注拦截原因，不进入审批 |
| **错误** | AI 系统异常（如识别失败） | 转人工录入兜底，不进入审批 |

## 五、业务规则一览

### 5.1 异常检测规则

以下情况会被系统自动拦截或预警：

| 检测项 | 规则 | 处理方式 |
|--------|------|---------|
| 字段缺失 | 发票号码/日期/金额/销售方/购买方为空 | **拦截** — 退回补全 |
| 格式错误 | 发票号码长度不符（8-20 位）或日期格式错误 | **拦截** — 退回修改 |
| 重复报销 | 同一发票号码 30 天内已报销过 | **拦截** — 阻止重复报销 |
| 票据过期 | 开票日期距申请日超过 180 天 | **拦截** — 提示票据已过期 |
| 日期异常 | 开票日期晚于申请日期 | **拦截** — 退回修改 |
| 金额异常 | 单张发票金额超过 10,000 元 | **拦截** — 需人工确认 |
| 申请金额不足 | 发票金额大于申请金额 | **拦截** — 退回修改 |
| 票据即将过期 | 票据剩余有效期不足 30 天 | **预警** — 提醒及时报销 |

### 5.2 费用分类限额

不同类型的费用有不同的报销上限，超出限额会标记预警：

| 费用分类 | 报销限额（元/单） |
|---------|-----------------|
| 餐饮 | 1,000 |
| 差旅 | 1,000 |
| 住宿 | 800 |
| 办公 | 500 |
| 交通 | 300 |
| 其他 | 200 |

> 以上限额可由系统管理员在规则配置文件中调整，无需修改程序代码。

### 5.3 审批权限阶梯

报销金额不同，审批级别也不同：

| 报销金额 | 审批人 | 备注 |
|---------|--------|------|
| 5,000 元以下 | 直属领导 | |
| 5,000 - 20,000 元 | 部门总监 | |
| 20,000 - 100,000 元 | VP / 分管副总 | |
| 100,000 元以上 | CEO | |
| 50,000 元以上 | 需两人会签 | 在对应级别基础上增加一位审批人 |

> 审批人可选择：通过、驳回（退回员工修改）、转审（交给同级或上级审批）。

### 5.4 行程单特殊规则

| 检测项 | 规则 |
|--------|------|
| 单笔行程金额异常 | 单笔超过 200 元标记异常 |
| 行程总金额异常 | 总金额超过 2,000 元标记异常 |
| 行程数异常 | 行程条数超过 50 条标记异常 |
| 日期合理性 | 行程日期不能早于开始日期或晚于结束日期 |
| 金额匹配 | 各行程明细金额之和需等于总金额 |

## 六、常见场景示例

### 场景1：正常餐饮报销（通过）

> 员工张三出差，上传一张 800 元的餐饮发票，申请金额 800 元。
>
> - AI 识别：提取发票号码、日期、金额 800 元、销售方信息
> - 异常检测：无重复、未过期、字段完整 → **通过**
> - 分类限额：餐饮类限额 1,000 元，800 < 1,000 → **通过**
> - 结果：自动进入直属领导审批 → 财务复核 → 发放

### 场景2：重复报销（拦截）

> 员工李四上传一张发票，该发票号码 30 天内已被报销过。
>
> - AI 识别：成功提取发票信息
> - 异常检测：查到该发票号码已在历史记录中 → **拦截**
> - 结果：直接退回员工，提示"该发票已于 X 月 X 日报销过"，不进入审批

### 场景3：费用超限（预警）

> 员工王五上传一张 1,200 元的餐饮发票。
>
> - AI 识别：成功提取发票信息
> - 异常检测：无异常 → **通过**
> - 分类限额：餐饮类限额 1,000 元，1,200 > 1,000 → **预警**
> - 结果：进入审批流程，报销单标注橙色"费用超限"标签，提醒审批人关注

### 场景4：小额免审

> 员工赵六上传一张 50 元的交通发票。
>
> - AI 识别：成功提取发票信息
> - 异常检测：无异常 → **通过**
> - 分类限额：金额 ≤ 100 元，**跳过限额校验**
> - 结果：直接进入审批流程

## 七、系统功能特性

| 特性 | 说明 |
|------|------|
| **多格式支持** | 支持 PDF、JPG、PNG 格式的发票和行程单上传 |
| **多文件批量上传** | 支持拖拽多选上传多张票据，系统并发处理 |
| **自动填单** | AI 识别后自动填充发票日期、金额、摘要、费用分类、发票号码、销售方名称、税额等字段 |
| **可视化校验过程** | 提交后前端展示 AI 逐步校验的动画（识别 → 异常检测 → 分类限额），每步可见 |
| **预警标识** | 报销单顶部醒目展示预警/拦截标签，审批人一眼可见风险 |
| **行程单智能校验** | 支持出租车行程单的 OCR 提取与合理性校验（金额匹配、天数、单笔最高、日期连续性） |
| **规则可配置** | 费用限额、异常规则、审批权限均通过配置文件管理，调整无需改代码 |
| **全链路审计** | 每次 AI 校验、数据修改、审批流转均记录日志，支持事后追溯 |

## 八、数据安全与合规

| 方面 | 措施 |
|------|------|
| 数据传输 | 全链路 HTTPS 加密传输 |
| 数据存储 | 票据文件加密存储（AES-256） |
| 敏感信息脱敏 | 手机号、银行账号等敏感字段脱敏展示（如 `138****1234`） |
| 权限隔离 | 员工只能看自己的报销单，领导看本部门，财务/管理员看全部（脱敏） |
| 审计追溯 | 全操作日志记录，审计日志不可删除 |
| 数据保留 | 报销凭证保留 ≥ 7 年（符合税务合规要求） |
| 数据生命周期 | 产生 → 活跃使用 → 归档（7年）→ 安全销毁 |

## 十、常见问题

**Q：AI 识别错了怎么办？**
A：系统在识别结果旁展示原始票据图片，支持人工对比复核。若 AI 完全无法识别，系统返回"错误"状态，转人工录入。

**Q：规则（限额/审批权限）需要调整怎么办？**
A：联系系统管理员，在配置文件中修改即可生效，无需开发人员修改代码。金额阈值调整需财务部门审批。

**Q：票据图片不清晰能识别吗？**
A：建议上传清晰的电子发票或扫描件。扫描件（无文本层）会自动转为图片识别模式。

**Q：被拦截了如何修改？**
A：系统会标注拦截原因（如"发票号码缺失""重复报销""票据过期"等），员工根据提示修改后重新提交即可。

**Q：审批人不在怎么办？**
A：审批人可选择"转审"，将审批权交给同级别或更高级别的审批人。
